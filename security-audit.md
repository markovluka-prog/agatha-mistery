# Security Audit

## Найденные риски

### Средний риск
- Формы полагались почти целиком на клиентскую валидацию.
- Не было даже базовой honeypot-защиты от простых ботов.
- Не было проверки слишком быстрого сабмита формы.
- Поиск фанфиков фильтровал не то поле (`character` вместо `genre`), что ломало часть поведения.

### Низкий риск
- В нескольких местах для UI-сообщений использовался `innerHTML`, хотя можно безопасно собирать DOM-узлы.
- Клиентский rate limit на `localStorage` полезен, но не является защитой сам по себе.
- Без серверного антиспама сайт всё ещё зависит от RLS и moderation flow в Supabase.

## Что исправлено сейчас
- Добавлена honeypot-защита для отзывов, фанфиков и иллюстраций.
- Добавлена проверка слишком быстрого сабмита формы.
- Успешные сообщения формы переведены на более безопасную сборку через DOM вместо строкового HTML.
- Исправлен фильтр поиска фанфиков по реальному полю `genre`.
- Добавлена серверная заготовка `supabase/functions/public-submit` для rate limit и валидации отзывов/фанфиков.
- Добавлена таблица `submission_rate_limits` в [supabase/secure-submit.sql](/workspaces/My-Projects/agatha-mistery/supabase/secure-submit.sql).

## Что стоит сделать дальше
